# Security policy

Security fixes are provided for the latest stable release and the active release candidate.

Do not open a public issue for vulnerabilities that could expose Home Assistant data, files, authentication details, or private configuration. Contact the maintainer privately through the contact method available on the maintainer's GitHub profile.

Dependency reports and screenshots may expose entity IDs, device names, automation names, rooms, addresses, locations, configuration-entry details, and relationships between security-sensitive entities.

Private reports are stored below `/config/entity_dependency_engine/reports/`. Optional public copies below `/config/www` may be accessible through Home Assistant `/local` paths. Public copies are disabled by default.
